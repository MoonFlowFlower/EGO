# Codex Autopilot Epic Rollup v1 SPEC

## Goal

Make Autopilot understand Epic overview issues as containers, not executable work:

- child issues carry an explicit `Parent epic: #N` marker;
- completed child sets can close their Epic overview with evidence;
- human-required child issues block Epic closeout;
- when no ready issue exists, Autopilot checks Epic rollup before creating generic first-slice proposals.

## Non-Goals

- Do not turn `Epic ` issues into ready executable issues.
- Do not modify `EgoOperator/**`, legacy projects, `docs/PROGRAM_STATE_UNIFIED.yaml`, or `artifacts/evidence_ledger/**`.
- Do not claim full unattended autonomous development, stable productivity gain, runtime efficacy, live autonomy, durable memory efficacy, or consciousness.

## Claim Ceiling

`Codex autopilot epic rollup and real-task execution local workflow candidate pass`.
