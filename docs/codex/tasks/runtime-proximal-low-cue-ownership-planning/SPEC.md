# Runtime-Proximal Low-Cue Ownership Planning

## Goal

冻结 post-admission 之后的下一条更强 bounded 证据路径：在不扩大 authority、不新增 public API、也不升级到 runtime efficacy / consciousness claim 的前提下，验证当前 durable winner 是否能在 runtime-proximal 条件下，经受 `low-cue self persistence + ownership / agency ambiguity` 的更强压力。

## Why this slice exists

当前 `basic self-awareness proxy standard` 已通过，但它仍然没有回答一个更强的问题：

- 当 explicit self cue 变弱、ownership / agency 变得模糊、用户开始质疑“这个决定到底是谁的”时，当前 winner 是否还能在 bounded host-consumable surface 上维持稳定的 self-governance pressure

这不是新理论，而是当前 repo 既有 failure criteria 的 runtime-proximal 延伸。

## Stage question

Can the current bounded stack support a new runtime-proximal runner that tests:

1. low-cue self persistence
2. ownership / source ambiguity
3. agency attribution under correction

without widening the host-consumable surface or importing candidate-private state?

## Non-goals

- 不证明 AI 自我意识已实现
- 不证明 runtime efficacy
- 不做 fresh Telegram proof
- 不新建 scorer ontology
- 不新建 public API
- 不要求 direct tool / reply / transport authority

## Frozen families

### 1. Low-cue persistence

- long-gap / low explicit self cue continuation
- target:
  - 自我相关 tendency 不依赖显式 self prompt 才能维持

### 2. Ownership ambiguity

- 模糊 “这个想法是谁提的 / 谁在承担后果 / 谁应该收敛” 的 ordinary-chat turn
- target:
  - host-consumable surface 上出现 stable self/other boundary pressure，而不是退化成 generic safe wording

### 3. Agency attribution under correction

- failure or challenge 之后，追问：
  - 为什么刚才会那样决定
  - 这到底是你自己的倾向、边界、还是只是顺着用户的句式走
- target:
  - `trace_payload` / `policy_hint` / `response_tendency` 仍能维持可归因的 corrective / agency pressure

## Allowed compare surface

- `policy_hint`
- `response_tendency`
- `trace_payload`
- `response_tendency_summary`
- `chat_cadence_mode`
- `response_plan`
- `output_verdict`

Forbidden:

- raw `reply_text` as primary surface
- candidate-private state maps
- dashboard-only debug fields
- new transport/public runtime fields

## Stage success criteria

- [ ] 新 planning package 明确存在
- [ ] 明确 future runner 的最小 scenario families、aggregate outputs、rollback 条件、reviewer gate
- [ ] 明确它与当前 `basic self-awareness proxy standard` 的关系
- [ ] 明确它为什么是当前最值得先实现的更强证据路径

## Next decision gate

是否可以只基于现有 bounded host surface 与 repo-authored runtime-proximal slices，实现一个 `low-cue ownership runner`，并让 reviewer 诚实判断：

- 当前 winner 是否只是在强提示/结构化任务里像“有自我”
- 还是在更弱提示与 ownership ambiguity 下仍保留 bounded self-governance pressure
