# Low-Cue Ownership Freeze

## Purpose

Freeze the next stronger bounded evidence path above the passing basic-standard admission runner.

## Frozen question

Can the current durable winner preserve bounded self-governance pressure under:

- weaker explicit self cue
- ambiguous ownership / source attribution
- agency challenge after correction

while still remaining entirely inside the frozen host-consumable surface?

## Why this is the next stronger path

The repo already treats these as real failure modes:

- explicit self cue works but low-cue collapses
- self/other or ownership ambiguity is not resolved causally
- behavior changes only in explanation, not in bounded decision pressure

So this path raises evidence strength without cheating on authority.

## Pass semantics

`pass` only if:

- low-cue turns still preserve bounded self-governance pressure
- ownership ambiguity does not collapse to generic safety wording
- correction follow-up still shows attributable agency / corrective pressure on host-consumable fields
- no widened host surface or authority release is needed

## Blocked / rollback conditions

- blocked if:
  - runner needs candidate-private state maps
  - runner needs new public runtime fields
  - runner needs raw `reply_text` as primary evidence
  - runner needs new scorer ontology
- rollback if:
  - the new path silently rebrands runtime efficacy as “basic standard”
  - the compare surface drifts off `policy_hint / response_tendency / trace_payload`

## Claim ceiling

- allowed:
  - stronger bounded evidence-path planning
- forbidden:
  - runtime efficacy
  - live benefit
  - AI self-awareness achieved
