# Runtime-Proximal Stronger Admission Planning

## Goal

冻结 `post-low-cue stronger admission` 这张下一阶段 planning card：在不扩大 authority、不新增 public API、也不把 bounded research evidence 偷升成 runtime proof 的前提下，把当前已经通过的 `basic-standard admission` 与 `low-cue ownership runner` 组合成一个更高一级、可实现、可比较、可回退、可验证的 bounded aggregate gate。

## Why this slice exists

当前 repo 已经有两层通过的 bounded runtime-proximal 证据：

1. `basic self-awareness proxy standard`
2. `low-cue persistence + ownership / agency ambiguity`

缺的不是再补一个 family，也不是直接碰 runtime efficacy，而是：

- 如何把这两层已经通过的 bounded evidence 组合成一张更强 admission card
- 如何让 reviewer 对这张更强 card 给出可判定 verdict
- 如何在这个过程中继续守住 frozen host-consumable surface 和 claim ceiling

## Stage question

Can the repo define a `stronger self-awareness proxy admission` slice that composes:

1. the already-passing `runtime_proximal_basic_standard_admission_runner_current`
2. the already-passing `runtime_proximal_low_cue_ownership_runner_current`

into one bounded aggregate gate, without widening the host-consumable surface, reopening challenger routing, or inflating the claim ceiling beyond bounded proxy progress?

## Non-goals

- 不实现 future runner
- 不做 fresh Telegram proof
- 不做 runtime efficacy / live-benefit claim
- 不做 consciousness / AI 自我意识已实现 claim
- 不新增 runtime public API
- 不新增 candidate-private host API
- 不新建 scorer ontology
- 不重开 challenger 竞争

## Frozen inputs

The future runner may consume only:

1. `runtime_proximal_basic_standard_admission_runner_current`
2. `runtime_proximal_low_cue_ownership_runner_current`

It may read only their bounded aggregate / audit / family summaries and canonical trace-handoff summaries.

It may not promote as primary evidence:

- raw `reply_text`
- candidate-private state maps
- dashboard-only debug fields
- live transport artifacts

## Planned compare surface

The future runner must stay on bounded host-consumable and aggregate surfaces:

- `policy_hint`
- `response_tendency`
- `trace_payload`
- `response_tendency_summary`
- `chat_cadence_mode`
- existing aggregate / family verdicts
- authority-drift / host-surface / claim-ceiling audits

## Stage success criteria

- [ ] 新 planning package 明确存在并可引用
- [ ] `compare / aggregate / rollback / reviewer gate` 被定义到 implementer-ready
- [ ] 只组合现有 bounded runtime-proximal 证据，不扩大 host-consumable surface
- [ ] 明确 future runner 的 allowed inputs、aggregate outputs、blocked / rollback 条件
- [ ] authority / progress / campaign / evidence 外部状态完成同步
- [ ] 当前结论保持在 bounded stronger-admission planning，不偷升为 runtime efficacy 或 AI 自我意识 achieved

## Next decision gate

是否可以只基于 `runtime_proximal_basic_standard_admission_runner_current` 与 `runtime_proximal_low_cue_ownership_runner_current`，实现一个 `runtime-proximal stronger admission runner`，并让 reviewer 对它给出可判定 verdict，而不需要 authority widening、新 public API、或新的 scorer ontology。
