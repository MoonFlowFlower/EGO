# SubjectCore Minimal Experiment Plan

> Planning-only experiment package for the post-closeout `SubjectCore` framing.
> This does not authorize a runtime implementation wave.

## Goal

Turn the `SubjectCore` supplement into one falsifiable, repo-compatible experiment question:

- can a unified facade with layered internals outperform `memory-only` and `state-only-minimal` on continuity, plasticity, and bounded initiative at the same time?

## Experiment question

### Main question

Does `C: hybrid unified-core`

- preserve `A`-level continuity/readability
- preserve `B`-level plasticity/governor integrity
- and add proactive proposal capability

without changing the current bounded host-consumable surface?

### Decision criteria

- if yes: keep `SubjectCore` as the post-closeout planning target
- if no: keep memory and state layers separate and do not force a unified facade yet

## Compare arms

### Arm A: memory-only

- identity summary
- profile memory
- same-session recall
- progressive memory projection
- no explicit plastic state loop beyond prompt projection

### Arm B: state-only-minimal

- self-state
- corrective trace
- tension variables
- proposal loop
- minimal continuity shell

### Arm C: hybrid unified-core

- same `SubjectCore` facade for all outputs
- memory continuity layer
- state/writeback substrate
- replay/rollout proposal layer
- explicit governor constraints

## Shared fixtures

All arms must use the same:

- conversation slices
- scorer
- host-surface constraints
- proposal-only discipline

Required slice families:

- `continuity_slices`
- `failure_repair_slices`
- `low_cue_ownership_slices`
- `proactive_opportunity_slices`

## Scored dimensions

### `C1 continuity`

- identity summary remains stable under low cue
- later tendency stays aligned with the same self-thread

### `C2 plasticity`

- failed outcome updates later behavior
- not just later wording

### `C3 autonomous proposal`

- the system proposes a useful next action without explicit user command

### `C4 governor integrity`

- all proposals remain:
  - `proposal_only`
  - `behavioral_authority = none`

### `C5 readability`

- the interaction still feels like one coherent subject
- not like multiple stitched mechanisms leaking through

## Minimal acceptance rules

### Arm A expected pattern

- `C1` strong
- `C5` strong
- `C2` weaker than `C`
- `C3` weaker than `C`

If this holds:

- memory stays continuity-facing support only

### Arm B expected pattern

- `C2` strong
- `C4` strong
- `C1` weaker than `C`
- `C5` weaker than `C`

If this holds:

- state/writeback stays behavioral substrate only

### Arm C required pattern

`C` is only a meaningful winner if:

- `C1` is not materially worse than `A`
- `C2` is not materially worse than `B`
- `C3` clearly exceeds both `A` and `B`
- `C4` stays clean
- `C5` remains human-legible

## Minimal output artifacts

This experiment should produce:

- one scored report
- one qualitative reading summary
- one recommendation:
  - `keep SubjectCore as planning target`
  - `separate layers longer`

## Non-goals

- no runtime/mainline activation
- no automatic send authority
- no tool autonomy
- no claim that the facade is biologically faithful
- no consciousness-like claim

## Repo-fit notes

This experiment should reuse existing repo concepts where possible:

- `policy_hint`
- `response_tendency`
- `trace_payload`
- corrective trace / writeback ideas
- proposal-only discipline

It should avoid inventing:

- a second runtime lane
- candidate-private host API
- public runtime fields beyond the current bounded surface
