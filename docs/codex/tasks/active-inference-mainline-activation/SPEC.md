> Current authority note: this spec started as the Stage 1 bootstrap package. For the current execution state and frozen tranche status, trust `docs/PROGRAM_STATE_UNIFIED.yaml` plus this task's `STATUS.md`, not the original bootstrap framing below.

## Goal

把 `active-inference self-model` 从已选中的 durable build-first research candidate，推进为 repo 当前唯一默认 implementation track，并以 `Stage 1 = unified-ingress ordinary-chat subject ingress stabilization` 作为第一执行切片。

## Non-goals

- 不在本任务内直接证明 `downstream tendency change / continuity effect`
- 不在本任务内直接证明真实用户收益、runtime efficacy、或 consciousness / AI self-awareness achieved
- 不扩 `host-consumable surface`
- 不释放 `direct tool / direct reply / direct transport authority`
- 不重开 `MVS-aligned compact` 或 `WP17/MVP22` 竞争

## Constraints

- 边界约束：
  - formal runtime mainline 继续唯一；`Telegram` 与 `dashboard chat` 只有在共享同一 unified ingress / formal runtime mainline 时，才属于同一 Stage 1 验收面
  - `EgoCore` 仍是宿主与现实裁决；`OpenEmotion` 仍是主体语义权威源
  - Stage 1 只允许落在 unified-ingress/runtime ingress 与 audit/accounting 面
- 仓库/子仓约束：
  - repo authority source 仍是 `docs/PROGRAM_STATE_UNIFIED.yaml`
  - 当前 diff 需要保持 scoped，不顺手推进 Stage 2 / Stage 3
- 环境约束：
  - 当前真实 Telegram fresh proof 不在本轮自动生成；本轮以 task/bootstrap、主链接线、审计 contract、定向回归为主
- 发布约束：
  - 无验证证据时不得把 bounded research evidence 说成 runtime proof

## Problem framing

- 当前问题表述：
  - selection closeout 已完成，但 repo 还停在“post-stronger frontier reframing”，没有切到真实通道闭环 program
- 归一化后的问题表述：
  - 先把 repo 的当前 execution owner 改成 `active-inference mainline activation`，然后修正/隔离 ordinary chat 的 subject-ingress accounting，使 Stage 1 能诚实度量 `unexpected_subject_miss`
- 为什么这个 framing 更适合当前任务：
  - 当前真正的瓶颈不是再堆一层 bounded summary，而是把 candidate routing 变成主线 program，并把 pre-runtime/policy intercept 与 ordinary-chat mainline miss 分开

## Unknowns to eliminate

- 当前 pre-runtime early return 在 fresh samples 中能否留下足够明确的 intercept metadata，供 Stage 1 审计隔离
- 把 repo execution owner 切到 `active-inference mainline activation` 后，还缺什么 fresh entrypoint-tagged live audit 才能判定 Stage 1 完成
