# EgoDesktop PSPC Visual Shim v0 Acceptance

## Acceptance

- The seven existing PSPC proposal-hint packets map to local visual scenarios.
- `warm_approach`, `cautious_boundary`, and `low_interrupt_care` produce visibly different presentation profiles.
- Unsafe packet variants are rejected.
- Mapper output contains no executable action/proposal/gate/memory/user-message/transport fields.
- Viewer demo controls read local visual shim state and do not use `sendChatTurn`.
- EgoOperator runtime, gates, memory, approval, transport, human-trial harness, and proactive paths are untouched.
- Claim ceiling remains `product_only_local_visual_behavior_mapping_from_shadow_proposal_hint`.
- Artifact report states what this proves, what it does not prove, failure meaning, and rollback.

## What This Proves

This proves existing PSPC proposal-hint artifacts can be converted into local, presentation-only Live2D visual demo state inside EgoDesktop without runtime authority.

## What This Does Not Prove

This does not prove PSPC runtime integration, adapter readiness, EgoOperator runtime efficacy, stable real user benefit, live autonomy, durable memory efficacy, consciousness, subjective experience, or real emotion.
