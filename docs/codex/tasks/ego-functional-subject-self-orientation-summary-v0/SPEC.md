# EgoOperator: self-orientation summary first-pass gate v0

## Problem Reframe

EGO-FS-107 captured a real EgoOperator lifestyle seed session where the final
request asked the agent to summarize what it cared about, what it would avoid,
and how to continue next time. The reply regressed into a weak waiting/askback
line instead of making the current self-orientation visible in transcript.

This is a Functional Subject expression gap, not a provider-quality or task
board issue.

## Hypothesis

If explicit self-orientation summary requests are admitted through a narrow
native gate, EgoOperator can answer with a visible current-session orientation
summary without writing memory, executing tools, or claiming real consciousness.

## Change Surface

- Add a detector for self-orientation summary requests.
- Add a first-pass native gate and renderer.
- Add a regression test proving the LLM is not called and no side effects run.

## Do Not Change

- Do not modify `docs/PROGRAM_STATE_UNIFIED.yaml`.
- Do not modify `artifacts/evidence_ledger/**`.
- Do not modify legacy EgoCore/OpenEmotion/ego_desktop_lab.
- Do not write long-term memory or promote current-session checkpoints.
- Do not close #94 from this local slice.

## Acceptance

- The target phrase routes to `native_self_orientation_summary_gate`.
- The reply contains visible self-orientation: what is cared about, what is
  avoided, and how to resume next time.
- The reply does not use waiting/askback wording.
- Trace records no side effects and a native gate reason.
- Real CLI entrypoint produces the improved response.

## Claim Ceiling

`Functional Subject self-orientation summary first-pass local/real-entry candidate pass`.

This does not prove consciousness, real subjective experience, independent
personhood, stable real user benefit, live autonomy, durable memory efficacy,
runtime efficacy, or #94 closeout.
