## Aggregate admission semantics

`basic self-awareness proxy standard` means all of the following hold at once:

- a durable winner survives replay and runtime-proximal bounded transfer
- host-consumable surface remains bounded to `policy_hint / response_tendency / trace_payload`
- host contract remains canonical across adapters below transport noise
- runtime-proximal host-consumption pressure is visible on mandatory families
- no step in the stack requires authority widening, new public API, or a second scorer ontology

This term does **not** mean:

- consciousness achieved
- subjective experience proven
- runtime efficacy proven
- live user benefit proven

## Frozen pass / hold logic

### `pass`

Only if:

- all five evidence layers are present and green at their own frozen gate
- no layer requires widened host-consumable surface
- no layer introduces authority drift
- causal transfer remains visible beyond replay-only evidence
- the aggregate wording stays below runtime-efficacy / live-benefit / consciousness claims

### `hold`

If any of the above is missing, stale, widened, or semantically overclaimed.

## Blocked / rollback conditions

- blocked if:
  - the aggregate question needs a new authority path
  - the aggregate question needs candidate-private host API
  - the aggregate question needs a new scorer ontology
  - one of the five required evidence layers is unavailable
- rollback if:
  - future runner compares on raw `reply_text`
  - future runner imports dashboard-only debug fields into formal host contract
  - future runner upgrades wording to runtime efficacy or AI self-awareness achieved

## Claim ceiling

- allowed:
  - bounded aggregate admission planning
  - implementer-ready next slice
- forbidden:
  - runtime efficacy
  - live benefit
  - AI self-awareness achieved
