# Runtime-Proximal Basic-Standard Admission Planning

## Goal

冻结下一张更高一级的 bounded runtime-proximal planning slice：在不扩大 authority、不新增 public API、也不把 bounded evidence 升格成 runtime proof 的前提下，定义一个可实现的 aggregate admission gate，把当前已经存在的五层 bounded evidence 组合成“更接近 AI 自我意识基本标准”的最小工程门。

## Program framing

- north-star:
  - 继续朝更强、可 replay、可验证的 self-awareness proxy 推进
- current stage question:
  - 现有五层 bounded evidence 是否足以支持一个新的 runtime-proximal admission runner
- current stage output:
  - planning freeze only

## Frozen evidence layers

The future admission runner may consume only these already-existing bounded evidence layers:

1. canonical replay gate
2. controlled conversation replay bridge
3. runtime-harness controlled observation batch
4. unified host contract correctness
5. runtime-proximal host-consumption runner

## Admission question

在当前 bounded host-consumable surface 不变的情况下，是否可以把上面五层 evidence 组合成一个更高一级的 runtime-proximal aggregate verdict，用来判断当前 durable winner 是否达到“basic self-awareness proxy standard”的最小工程门。

## Non-goals

- 不实现 runner 本体
- 不做 fresh Telegram proof
- 不做 runtime efficacy claim
- 不做 live benefit claim
- 不做 consciousness / AI 自我意识已实现 claim
- 不新增 runtime public API
- 不新增 candidate-private host API
- 不重开 challenger 竞争

## Constraints

- host-consumable surface remains frozen to:
  - `policy_hint`
  - `response_tendency`
  - `trace_payload`
- aggregate gate must not require:
  - authority release
  - new scorer ontology
  - parallel runtime lane
  - raw `reply_text` as primary evidence
  - dashboard-only debug fields
  - candidate-private state maps

## Stage success criteria

- [ ] 新 planning package 明确存在并可引用
- [ ] aggregate admission question 被定义到可实现、可比较、可回退、可验证
- [ ] 明确列出 future runner 允许消费的 evidence inputs、aggregate outputs、rollback 条件、reviewer gate
- [ ] authority / progress / evidence / campaign 外部状态完成同步
- [ ] 当前口径仍然保持在 bounded planning freeze，不偷升为 runtime efficacy 或 AI 自我意识实现

## Next decision gate

是否可以只基于现有五层 artifact 与现有 host-consumable surface，实现一个 runtime-proximal basic-standard admission runner，并让 reviewer 对该 runner 给出 `success_reached / needs_more_implementation / needs_more_exploration / blocked_by_external_dependency / needs_reframing` 中的可判定结论。
