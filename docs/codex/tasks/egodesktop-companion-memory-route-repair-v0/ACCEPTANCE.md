# Acceptance

## Positive Acceptance

- EgoDesktop can answer session-local recall questions in companion-visible language without exposing engineering policy text.
- With a valid window-lifetime session context, questions like `还记得我喜欢什么奶茶吗` can cite actual same-window facts.
- With no session context, recall wording must not fabricate durable memory.
- Ordinary affectionate turns such as `喜欢你所以就摸摸你`, `像小猫一样`, and `摸摸头` must not be classified as creative/adult creative recovery.
- Timeout fallback wording must be route-neutral by default and must not say `创作路线超时` for generic backend timeout.
- PSPC reply preview remains local-only and keeps `runtime_authority=none`, `enabled=false`, and `mainline_connected=false`.

## Forbidden

- No EgoOperator core memory/gate/approval/transport/proactive changes.
- No real memory write.
- No PSPC adapter creation or registration.
- No user-visible claim of durable memory unless an explicit approved memory write exists.
- No `candidate-local`, `operator memory`, `PROJECT_MEMORY`, `program state`, `evidence ledger`, or `memory approval` in ordinary EgoDesktop companion replies unless the user explicitly asks about the engineering mechanism.

## What This Proves

This can prove only that EgoDesktop local visible replies and local timeout recovery can preserve the companion experience without widening runtime authority.

## What This Does Not Prove

This does not prove PSPC mainline integration, durable memory, true learning, stable real user benefit, live autonomy, consciousness, subjective experience, or real emotion.
