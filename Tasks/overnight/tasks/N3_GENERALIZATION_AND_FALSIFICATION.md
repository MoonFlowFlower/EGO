# N3 — 泛化、误聚合与反证样本

## 任务类型
观察验证 / 泛化测试 / 风险暴露

## 目标
确认当前 Proto-Self Kernel v1 的有效性不是只建立在少数样本上，并暴露 coarse intent bucket 的误聚合边界。

## 当前层级
泛化验证层

## 当前确定项
- cycle 聚合曾经过细，后修成粗粒度 intent bucket
- 当前主线已通过真实 Telegram E2E

## 关键未知
- coarse intent bucket 是否会误聚合不同真实意图
- replay 是否足以稳定重放关键信息
- 更多真实样本下是否仍能保持相似行为

## 成功判据
1. 至少收集或构造一组“应聚合”样本
2. 至少收集或构造一组“应区分”样本
3. 对两组样本分别给出：
   - 输入
   - bucket / cycle 命中情况
   - trace 或状态变化
   - 是否符合预期
4. 至少一组 replay / trace 一致性检查
5. 明确列出：
   - 已证实范围
   - 未证实范围
   - 已发现风险

## 硬约束
- 不允许为通过而把样本写得太像
- 不允许忽略失败样本
- 不允许把“暂未发现问题”写成“问题不存在”
- 不允许修改主链行为只是为了掩盖误聚合

## 允许修改
- 对比脚本
- replay 检查脚本
- 观测脚本
- 测试样本集

## 禁止修改
- 大幅改写生产 cycle 逻辑（除非 N3 被显式升级为修复任务）
- 篡改历史 artifact
- 无证据更新正式成功口径

## 必交付物
- 样本集
- 结果对比表
- 风险说明
- `reports/N3_REPORT.md`
- `runtime/RUN_STATE.json` 更新

## 必须记录的证据
- 命中的 cycle / bucket
- 是否误聚合
- replay 一致性结果
- 反例与边界说明

## 失败出口
若发现明显误聚合：
- 本任务可以 `verified` 风险识别，但不能宣称主链完全稳固
- 报告必须转入“下一轮修复建议”，而不是偷偷掩盖
