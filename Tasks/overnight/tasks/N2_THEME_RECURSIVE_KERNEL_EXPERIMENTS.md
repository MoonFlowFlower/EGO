# N2 主题 — 递归核有效性实验主题

## 任务类型
实验主题批次

## 目标
把“递归核是否有效”从聊天直觉推进成可复现、可反驳、可比较的实验结果。

## 当前层级
理论可行性验证层

## 本主题允许连续推进的子任务
- N2A 实验合同冻结
- N2B 实验脚手架与 artifact 通道
- N2C 主实验运行
- N2D Ablation 与对比实验
- N2E 主题汇总与结论报告

## 主题级成功判据
1. 至少形成一套实验合同
2. 至少有可运行的实验脚手架
3. 至少跑完一组主实验
4. 至少给出一组 ablation 或明确写明无法进行的原因
5. 输出统一主题报告，明确：
   - 已证实范围
   - 未证实范围
   - 反例与失败条件

## 主题级硬约束
- 不要把实验代码混成生产主链
- 不要为了让实验通过而绕过治理壳
- 不要宣称“意识已证明”
- 不要把看起来像当通过标准

## 主题级失败出口
若主实验或 ablation 发现递归核当前无法支撑某些成功判据：
- 不要硬修成漂亮结果
- 直接记录反例与失败条件
- 本主题允许以 `partial` 收口，但必须留下清楚的反证
