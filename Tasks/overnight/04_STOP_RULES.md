# 强制停止条件

出现以下任一情况，立即停止当前子任务；必要时停止整个主题，并写明阻塞原因。

## 一、当前子任务必须立即停

1. 当前任务的权威源不明确
2. 当前任务连续两次同层修补仍失败
3. 需要新增未授权的大范围改动
4. 依赖子任务未通过 Gate B 或 Gate C
5. 需要人工凭证、人工确认、外部账号交互
6. 运行环境异常，导致验证结果不可信
7. 样本先被安全层拒绝，导致实验结论失真

## 二、当前主题必须立即停

1. 核心前置子任务 blocked
2. 当前主题开始跨到别的主题才能继续
3. 文件交接状态与实际状态冲突
4. artifact / report / run_state 三者互相不一致
5. 已经无法判断“当前结论来自真实证据还是会话幻觉”

## 三、允许停在 partial 的情况

以下情况不必硬做成 verified：

- 实验已发现反例
- 脚手架已完成，但真实样本还不足
- 已实现只读诊断壳，但尚未完成真实用户体验验证
- 已暴露风险边界，但修复不属于当前主题

这种情况下必须明确：
- 已做了什么
- 证据是什么
- 缺什么才到下一状态
- 最小回退动作是什么
