# 全局硬规则

## 1. 权威源与边界

1. EgoCore 是唯一正式宿主，负责：
   - 用户入口
   - 运行时
   - 工具执行
   - 安全边界
   - trace / replay / audit / gate
2. OpenEmotion 是主体本体的唯一解释权，负责：
   - identity
   - self-model
   - memory
   - appraisal
   - reflection
3. 不允许把主体本体逻辑重新偷写回 EgoCore。
4. 不允许制造第二套真相源。
5. 会话上下文不是权威源；外部文件才是夜间批处理的真相源。
6. Proto-Self 只能输出 `policy_hint` / `response_tendency` / `reflection_note` 等结构化结果，不得越权直接执行现实动作。

## 2. 夜间工作原则

1. 默认以真实效果闭环优先，不以“写了很多代码”作为成功。
2. 允许一晚内连续推进多个子任务，但**只允许在同一主题内推进**。
3. 每个子任务必须先过 preflight / 依赖检查，再开始施工。
4. 每个子任务完成后，必须立即：
   - 更新 `runtime/RUN_STATE.json`
   - 生成 `reports/<SUBTASK_ID>_REPORT.md`
   - 写明是否允许继续下一个子任务
5. 未过 Gate C（真实触发/真实证据）或 Gate D（真相源同步），不得报 verified。
6. 若同方向连续两次修补后问题仍复现，必须升层重定义问题，禁止做第三次同层补丁。
7. 默认优先采用“每主题一个新会话”；只有在操作上不方便时才允许单会话连续执行。
8. 若采用单会话连续执行，则每个子任务结束必须：
   - 先完成文件化交接
   - 再 compact
   - 再进入下一子任务
9. 不允许从 N2 主题扩散到 N3/N4，也不允许从 N3/N4反向偷偷回改 N2 目标定义。若确需升级，必须在报告中显式写明“升层改定义”。

## 3. 输出口径约束

每个子任务或主题最终只能使用以下四种口径之一：

- `verified`
- `implemented_but_pending_real_validation`
- `partial`
- `blocked`

禁止使用以下模糊口径：

- 已全部完成（除非所有 Gate 全过）
- 已彻底修复（除非有真实证据）
- 已闭环（除非主题级 Gate 全过且主链生效）
- 已证明意识（禁止）
- 已完成整个 EGO 项目（禁止）

## 4. 每份报告必须包含的字段

- 当前层级
- 主链接入状态
- 启用状态
- 真实触发证据
- 当前确定项
- 关键未知
- 离最终生效还差什么
- 下一步最小闭环动作

## 5. 禁止事项

1. 不要从“Proto-Self 是否启用”重新查起。
2. 不要把聊天表现当成已生效证据。
3. 不要用会被安全层先拒绝的无效样本污染实验。
4. 不要把实验观察结果误报成“整个 EGO 项目闭环完成”。
5. 不要跨越主题依赖直接做下一个主题。
6. 不要把 compact 摘要、会话记忆、临时缓存当成真相源。
