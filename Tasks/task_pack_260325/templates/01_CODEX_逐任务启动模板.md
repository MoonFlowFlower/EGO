# 给 Codex 的逐任务启动模板

把下面整段复制给 Codex，并替换方括号内容。

---
仓库：`https://github.com/pen364692088/EGO`

你现在只做 **一个任务**：
`[任务文件名，例如 P1_RuntimeV2Loop_主链瘦身手术.md]`

强制要求：
1. 先完整阅读：
   - `00_README.md`
   - `01_MASTER_EXECUTION_ORDER.md`
   - `tasks/[任务文件名]`
2. 先输出：
   - 任务类型
   - 目标与成功判据
   - 当前层级
   - 当前确定项
   - 关键未知
   - 你准备采用的唯一主执行链
3. 然后在仓库内完成该任务，过程中遵守：
   - 先审计再改造
   - 不扩大范围
   - 不顺手做下一个任务
   - 不要把猜测写成事实
4. 结束时必须提交以下产物：
   - `artifacts/[任务编号]/TASK_REPORT.md`
   - `artifacts/[任务编号]/CHANGE_PLAN.md`
   - `artifacts/[任务编号]/EVIDENCE_TABLE.md`
   - `artifacts/[任务编号]/FAILURE_SAMPLES.md`
5. 结束时必须明确写出：
   - 当前层级
   - 主链接入状态
   - 启用状态
   - 真实触发证据
   - 当前确定项
   - 关键未知
   - 本次结论不能证明什么
   - 下一步最小闭环动作

禁止事项：
- 禁止同时做多个任务
- 禁止把未验证内容写成已完成
- 禁止因为上下文长就跳过审计
- 禁止扩大到无关模块大修
---

如果你发现该任务前提不成立，停止施工，直接输出《前提失效报告》。
