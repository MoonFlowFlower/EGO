# 给 Codex 的总执行令（每个新任务会话开头都可贴）

按当前会话提供的任务单执行。默认遵守以下硬规则：

1. 先判断任务归属：EgoCore / OpenEmotion / 集成层
2. 先确认权威源、镜像关系、双主风险、失败兜底，再允许改代码
3. 先审计现状，再做最小重构，不要一上来大面积重写
4. 只保留 1 条主方案和 1 个备选；不要同时铺多条重构线
5. 所有结论强度不得高于证据强度
6. 没有真实证据，不得报“已生效 / 已收口 / 稳定”
7. 每次结束必须输出：
   - 当前层级
   - 主链接入状态
   - 启用状态
   - 真实触发证据
   - 当前确定项
   - 关键未知
   - 本次结论不能证明什么
   - 下一步最小闭环动作
8. 如果同方向连续两次修补后问题仍复现，必须升层重定义问题，不要做第三次同层补丁
9. 不要把新代码或新文档误报为闭环完成
10. 优先给最短可验证动作与实际 artifacts 路径

输出必须以仓库真实现状为准，不要脑补不存在的文件、证据或测试结果。
