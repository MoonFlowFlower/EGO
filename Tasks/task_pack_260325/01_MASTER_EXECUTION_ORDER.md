# MASTER_EXECUTION_ORDER

## 当前目标
把 EGO 从“能跑 + 文档较多”收口成“边界干净 + 证据可信 + 可持续重构”的双核工程系统。

## 成功判据
- README / 验收报告 / 代码 / 真实证据之间不再互相打架
- `RuntimeV2Loop` 不再承担过多跨层责任
- 主体状态、trace、evidence 的权威源与镜像关系清楚
- shim / mirror / compatibility-only 都被登记并有退出计划
- 每项重构都能说明：改了什么、没改什么、靠什么证明

## 停止条件
出现以下任一情况必须停下并收口，不得继续串做后续任务：
1. 当前任务的权威源无法明确
2. 当前任务已经触及另一个任务的主问题
3. 当前任务两次修补后仍复现
4. 当前任务没有最小证据包，无法继续报完成
5. 当前任务导致 simulated / integration / real_channel 任一层回归破坏而未解释

## 任务节奏
- 每个任务独立会话
- 每个会话只做一个任务
- 每个任务最多允许 2 轮修补
- 第 3 次还没稳住，必须升级为新的“问题重定义任务”
